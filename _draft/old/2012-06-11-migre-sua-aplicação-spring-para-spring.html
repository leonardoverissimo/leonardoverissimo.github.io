--- layout: post title: Migre sua aplicação Spring para Spring tags: - Sem categoria status: draft type: post published: false meta: _edit_last: '1' --- Você já deve ter visto em blogs boas recomendações sobre o Java EE, dizendo que, para uma certa gama de aplicações, a utilização de CDI e EJB pode substituir com razoável facilidade funcionalidades hoje presentes no Spring. A justificativa para mudanças é sempre o bordão ideológico de que "os componentes Java EE foram ratificados por um comitê padronizador", ou coisa parecida. Minha recomendação ao olhar notícias assim: cuidado, existem interesses excusos por trás. Por mais que exista uma comunidade ativa de desenvolvedores Java e grandes empresas consumidoras da plataforma Java, ela é uma plataforma nas mãos de poucas vendors, que se reduzem mais e mais a medida em que ocorrem fusões. Pra que você entenda se o abandono de Spring vale a pena é preciso entender duas coisas: quem são os players do mercado e quem propagandeia a necessidade de abandono do Spring.
