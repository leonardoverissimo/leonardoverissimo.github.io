Classes são alteradas dinâmicamente tags: - Sem categoria status: draft type: post published: false meta: _edit_last: '1' --- Para um javeiro, é difícil admitir que alterar uma classe "a quente" seja realmente uma vantagem. Acostumado com a rigidez e a sensação de segurança da linguagem Java, qual seria a vantagem de, depois de criar uma classe, mudar para algo diferente? É, só-que se quem alterasse a classe não fosse você?, mas uma biblioteca que está fazendo isso no seu lugar? Isso é o que as gems de persistência como ActiveRecord, DataMapper, CouchRest. No caso do famoso ActiveRecord, uma classe do modelo poderia ficar assim: 
class Album < ActiveRecord::Base
  has_many :tracks
